ThousandWords

A node-linked Visual Creator Application supporting builtin nodes, scripted nodes, live nodes and more.

The underlying reason is to keep a simple-to-read library of graphics code. For that reason I strongly suggest you pattern your nodes on the examples, and don't obfuscate your code. Don't provide only binaries (you'd have to modify the core to do that). Don't charge, if you can resist the urge.

For example, an input node could have a cat picture. The output from that node might be fed into a script node to create a very specific effect. The output from the script node could be set to a frames node. The frames node might link bank to one of the parameters in the effect (script) node, allowing it to adjust over time.

A possible related repo : An output node code generate Processing code, although that would require some sort of C# -> Java transpiler.

If there's a slow but easy version of your plugin, and a lightning-fast but hard to read version, possibly include both, with a switch parameter.
